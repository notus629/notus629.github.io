搭建开发环境
   
    官方提供了 weex-toolkit 的脚手架工具来辅助开发和调试



首先，安装 Node.js 和 Weex CLi


    安装 Node.js 

        官方下载可执行文件安装即可

        使用以下命令检测是否安装成功

$ node -v
v6.3.1
$ npm -v
3.10.3


    安装 Weex CLi

        安装了 Node.js 环境，npm 包管理工具也随之安装了

        使用 npm 来安装 weex-toolkit （速度很慢)
            $ npm install -g weex-toolkit
            ## weex-toolkit 在 1.0.1 之后才支持初始化 Vue 项目，使用前请确认版本是否正确。
            $ weex -v

        国内推荐使用淘宝的 npm 镜像 —— cnpm 安装 weex-toolkit
            $ npm install -g cnpm --registry=https://registry.npm.taobao.org
            $ cnpm install -g weex-toolkit



Windows 10 安装遇到的问题

    $ cnmp install -g weex-toolkit 安装完成后，$ weex -v 没有问题，而 $ weex, 会出现 "Error: resolve path error". [1] 错误现象完全和 [2] 一样。

    若使用 $ npm install -g weex-toolkit 也会报错 npm ERR 类似的错误

    原因：主要是 npm 默认安装目录权限不足的问题。

    
    解决方法 [2]

        ## 先卸载 weex-toolkit
        $ npm uninstall -g weex-toolkit

        ## 查看 npm 和 node 的一些配置路径 
        $ npm config ls
        ## 修改 prefix 路径为自己指定的路径（最好不要有中文和空格）
        $ npm config set prefix "I:Nodejs ode_modules"

        ## 再使用 npm 安装 weex-toolkit
        $ npm install -g weex-toolkit
 
        ## 最后把环境变量设为 prefix 指定的路径即可安装成功。
        ## 到最后还是会有两个 npm WARN 的警告，不过可以正常的使用了，使用 $ weex -v 和 $ weex 没有问题
