dede 数据表核心分析
    
    a. dede 设计者认为：所有的栏目（类别）都应该属于某个模型 [比如:武侠文章/音乐文章], 此模型信息放在 dede_channeltype 模型表。（频道类型, 即模型）

    默认有 5 种模型可选择，若在实际开发中，这些模型不能满足需求时，可以自己增加模型。 


    b. dede 设计者认为：所有具体一篇文章/商品/软件/新闻都应该属于某个栏目，因此设计了一张表，专门用于保存栏目（类别信息）.即 dede_arctype 栏目表。（文档类型，即栏目）

    与 dede_channeltype 表的关联关系，由 dede_arctype 的 channel_type 列与 dede_channeltype 表的 id 对应。


    c. dede 设计者认为，不管是文章、商品、软件或新闻，他们都有共同的信息（比如标题、编号、点击率等），所以把共有的字段做成一张表，即 dede_archives 文档主表。

    dede_archives.type_id 与 dede_arctype 的 id 相对应。channel 对应 dede_channeltype 的 id


    d. dede 设计者认为，不管是文章、商品、软件、新闻，他们除了有相同的信息外(放入dede_archives表)，也有不同的信息（比如，电影之导演，商品则没有），就给每个模型生成一个对应的附加表。表名规则是：dede_addonXXX 文档附加表。基中 xxx 表示模型。

    如 dede_addonarchives, 其 aid 字段对应 dede_archives 的 id, 其 typeid 对应 dede-arctype 的 id.
