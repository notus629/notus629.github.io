本节课大纲：

一、URL规则
       1、默认是区分大小写的
       2、如果我们不想区分大小写可以改配置文件
               'URL_CASE_INSENSITIVE'=>true,//url不区分大小写        
       3、如果模块名为 UserGroupAction
               那么url找模块就必要要写成: user_group
               http://localhost/thinkphp4/index.php/user_group/index
       4、如果'URL_CASE_INSENSITIVE'=>false
               那么url也可以写为(user_group好像也可以)
               http://localhost/thinkphp4/index.php/UserGroup/index

二、URL伪静态
       默认设置了伪静态：可以使用.html, .pdf, .xml
       'URL_HTML_SUFFIX'=>'html|shtml|xml',//限制伪静态的后缀

三、URL路由
       使得可以用其它方式访问URL
       必须为pathinfo方式
       1、启动路由
               要在配置文件中开启路由支持
               'URL_ROUTE_ON'=>true,
       2、使用路由
               'URL_ROUTE_RULES' => array(  ),        //定义路由规则
           a. 规则表达式配置路由
               'my'=>'Index/index',                                        //静态地址路由, 使用my就可以访问Index模块下的index方法
               ':id/:num'=>'Index/index',                                //动态地址路由， 这样可以用100/1000访问，用于get传参, $_GET['num'], $_GET['id'];
               'my/:num'=>'Index/index',                                //混合地址路由, 可以用my/abc, $_GET['num']传值为'abc'
               'year/:year/:month/:date'=>'Index/index',                //动态和静态混合地址路由, echo "{year: $_GET['year'] }, {month: $_GET['month']}, {date: $_GET['date']}"
               加上 \d代表类型只能是数字（目前thinkphp只支持数字检测）
                       'year/:year\d/:month\d/:date\d'=>'Index/index',
               'my/:id$'=>'Index/index',// 加上$说明地址中只能是 my/1000 后面不能有其他内容了
           b. 正则表达式配置路由
               以/开始，以/结束
               转义符：\, 如\/
               \d{4}，数字出现4次
               ：1，第一部分，即\d{4}代替:1，此时须用括号括起
               ^year, 以year开始
                       '/^year\/(\d{4})\/(\d{2})\/(\d{2})/'=>'Index/index?year=:1&month=:2&date=:3'

       3、注意事项：
               a.  越复杂的路由越往前面放
                       'URL_ROUTE_RULES'=>array(
                               'my/:year/:month:/:day'=>'Index/day',
                               'my/:id\d'=>'Index/index',
                               'my/:name'=>'Index/index',
                               )
               b.  可以使用$作为完全匹配的路由规则
                       'URL_ROUTE_RULES'=>array(
                               'my/:id\d$'=>'Index/index',
                               'my/:name$'=>'Index/index',
                               'my/:year/:month:/:day$'=>'Index/day',
                               ),
               c.  用正则匹配的方式
                       'URL_ROUTE_RULES'=>array(
                               '/^my\/(\d+)$/'=>'Index/index?id=:1',
                               '/^my\/(\w+)$/'=>'Index/index?name=:1',
                               '/^my\/(\d{4})\/(\d{2})\/(\d{2})$/'=>'Index/day?year=:1&month=:2&day=:3',
                               ),
               
四、URL重写
       rewrite重写，即可以省略index.php
               Apache的配置过程，可以参考下：
                       1、httpd.conf配置文件中加载了mod_rewrite.so模块
                       2、AllowOverride None 将None改为 All
                       3、确保URL_MODEL设置为2        (可省）
                       4、把下面的内容保存为.htaccess文件放到入口文件的同级目录下
                               <IfModule mod_rewrite.c>
                               RewriteEngine on
                               RewriteCond %{REQUEST_FILENAME} !-d
                               RewriteCond %{REQUEST_FILENAME} !-f
                               RewriteRule ^(.*)$ index.php/$1 [QSA,PT,L]
                               </IfModule>
五、URL生成
       U方法
           U('[分组/模块/操作]?参数' [,'参数','伪静态后缀','是否跳转','显示域名'])
               U(Index/index)                //生成Index模块的index操作的URL地址
               U('Blog/read?id=1')        // 生成Blog模块的read操作 并且id为1的URL地址
               U('Blog/cate',array('cate_id'=>1,'status'=>1))        //
