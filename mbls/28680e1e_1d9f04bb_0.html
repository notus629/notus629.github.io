git checkout
    
    会覆盖当前的本地更改，所以更换分支前 git 强制你彻底放弃或提交存储当前的更改。

    git checkout 不会废弃任何分支或提交


    ## 切换 HEAD 到不同的分支，并且更新当前的 working directory 去匹配
    $ git checkout hotfix                  

    ## checkout 到任何一次 commit
    ## 如果再次添加新的提交就无法返回原先的状态。因此，你应该在修改前总是创建一个新的分支。
    $ git checkout HEAD~2 [commit id]      

    ## 放弃工作区对 readme.txt 文件的修改
    ## 若 readme.txt 自动修改后，还没有放到暂存区，会改为和版本库一模一样的状态。
    ## 若 readme.txt 已经放入暂存区了，接着又作了修改，撤销修改就回到添加暂存区后的状态。
    $ git checkout -- readme.txt
