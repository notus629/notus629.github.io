Remote Access error 623

    详细描述： Remote Access error 623 - The system could not find the phone book entry for this connection.

    错误原因：
        因为使用的是cmd的命令进行连接，之前将 cmd 控制台改成了 utf-8编码，从而找不到“VPN连接”的正确名字而导致。
    
    解决办法：
        改回 gbk 编码即可,cmd 中使用 chcp 936 可以换回默认的 GBK



VPN 错误 789

    问题描述：vpn 错误789:l2tp 连接尝试失败,因为安全层在初始化与远程计算机的协商时遇到一个处理错误

    Windows配置VPN，选择“使用IPsec的第2层隧道协议(L2TP/IPSec)”时，XP 系统肯定会报错，Win7 系统正常，Win8 系统大部分情况报错，有时正常。

    错误信息：错误789:l2tp 连接尝试失败,因为安全层在初始化与远程计算机的协商时遇到一个处理错误


    解决方法：                                          

        修改注册表来实现

        1. 单击“开始”，单击“运行”，键入“regedit”，然后单击“确定”

        2. 找到下面的注册表子项，然后单击它： HKEY_LOCAL_MACHINE\ System\CurrentControlSet\Services\Rasman\Parameters

        3. 在“编辑”菜单上，单击“新建”->“DWORD值”

        4. 在“名称”框中，键入“ProhibitIpSec”

        5. 在“数值数据”框中，键入“1”，然后单击“确定”

        6. 退出注册表编辑器，然后重新启动计算机
