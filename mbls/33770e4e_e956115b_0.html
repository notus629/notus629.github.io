几个路径函数

    通常目录前若可以有/，则会有/，而目录后则没有/

    getcwd();

        查看当前目录(current working directory)
        即当前执行脚本的路径，而非此函数所在脚本的目录。

        比如a.php中有语句echo getcwd(); 被 include 到 index.php中执行，则输出为: 
        C:\Program Files (x86)\Apache Software Foundation\Apache2.2\htdocs\phppractice\projects\shoppingcart （即index.php所在目录）


    chdir(...);                        // 改变当前目录，对img 的相对路径无效


    $_SERVER['DOCUMENT_ROOT']

        服务器根目录，即 C:\Program Files (x86)\Apache Software Foundation\Apache2.2\htdocs


    $_SERVER['SCRIPT_NAME']

        当前执行脚本的路径，去掉服务器根目录。
        如 index.php 包含 a.php, 在 a.php 中打印 echo $_SERVER['SCRIPT_NAME'];

        则为：/phppractice/projects/shoppingcart/index.php  // index.php 的路径


    $_SERVER['SCRIPT_FILENAME']

        当前执行脚本的绝对路径
        如 index.php 包含 a.php, 在 a.php 中打印 echo $_SERVER['SCRIPT_FILENAME'];
        
        则为：C:/Program Files (x86)/Apache Software Foundation/Apache2.2/htdocs/phppractice/projects/shoppingcart/index.php


    $_SERVER['SERVER_NAME']

        当前执行脚本所在的服务器主机名，即 localhost


    $_SERVER['SERVER_ADDR']

        当前执行的脚本所在的服务器的IP地址, 即 127.0.0.1   
    
    $_SERVER['PHP_SELF']

        当前执行的脚本相对于 document root 的文件名
        如 index.php 包含 a.php, 在 a.php 中打印 echo $_SERVER['PHP_SELF']; ， 输出：
            /phppractice/projects/shoppingcart/index.php


    $_SERVER['PATH_TRANSLATED'];

        默认没有值


    __DIR__                             

        指向当前执行的 PHP 脚本所在的目录 
        不随执行脚本而变化（似乎不起作用？？）


    __FILE__                            

        contains the full path and filename of the current
        不随执行脚本而变化
