什么是 MX 记录 [1]

    用于电子邮件系统发邮件时根据收信人的地址后缀来定位邮件服务器。

    例如，当有人发邮件给“vincen@example.com”时，系统将对“example.com”进行DNS中的MX记录解析。如果MX记录存在，系统就根据MX记录的优先级，将邮件转发到与该MX相应的邮件服务器上。

    MX 用于传递地址指向特定域名的电子邮件。与 CNAME 记录一样，MX 记录将一个域名或子域名指向其中存在 A 记录的另一个域名或子域名。


    举例

        邮箱：bugber@bugber.com

        会根据 bugber.com 所在的 DNS，找到对应的 MX 记录，确定 MX 记录中的 bugber.com (即 name 为 @)的值（value）为 mxbiz1.qq.com (Priority: 5) -  可能存在多个（?? priority 的大小关系）-，然后转而由 mxbiz1.qq.com 服务器来处理。



A (主机) 记录

    A 记录（通常也称为主机记录）, 即将某个域名或子域名指向某个 IP 地址。

    如主机 @， 指向 123.56.160.54， 即表示当前域名 bugber.com 指向 123.56.160.54

    主机 www, 指向 123.56.160.54, 表示 www.bugber.com 也指向 123.56.160.54



AAAA（IPv6）主机

    AAAA 记录存储着给定域名或子域名的 IPv6 地址。这些记录遵循标准的冒号分隔格式。



CNAME 记录

    CNAME 记录（也称为别名记录）将子域名指向已存在的 A（主机）记录，以便子域名解析到相应的 IP 地址。

    如，主机 ftp 指向 @, 即表明 ftp.bugber.com 指向 bugber.com
    
    主机 www 指向 @, 表明 www.bugber.com 指向 bugber.com



转址

    通过转址功能，您可以将自己的域名访问者转到另一个网站。

    子域名转址还可以在域名商添加前缀，从而将访问者重定向到其他 URL。例如，如果您的域名是 coolexample.com，那么您可以添加 subdomain.coolexample.com 并将它指向特定的 URL。

    不同于上面的其它记录，转址设置并不在 DNS 区域文件中。
