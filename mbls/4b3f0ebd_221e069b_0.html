事件流定义

    事件发生的顺序，即事件流

    如，把鼠标移动到一个链接上，或者单击此链接，同样会移动到它的父元素上，或单击它的父元素。



两种事件流顺序

    事件冒泡，即从最具体的节点到最一般的顺序(从子元素到包含它的父元素)，如

        <a>, <li>, <ul>, <body>, <html>, document 对象, window 对象


    事件捕获，事件从最宽泛的节点开始向内传播到最具体的节点。

        与事件冒泡相反，IE8 和更早版本的 IE 中不支持


    对于传统的 DOM 事件处理程序及 HTML 事件属性，所有的现代浏览器默认都会使用事件冒泡模型。


    对于 addEventListener，最后一个参数默认为 false (即冒泡方式），若设为 true, 则表示事件捕获方式。
