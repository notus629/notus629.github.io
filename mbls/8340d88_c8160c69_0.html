Memcached 服务

    分布式 memcached   


    基于 libevent 事件处理


    内置内存存储方式

        内存容量达到指定值后，就会使用 LRU （least recently used 最近最少使用）算法，删除不使用的缓存


    基于客户端的分布式

        memcached 尽管是"分布式"缓存服务器，但服务器端并没有分布式功能。各个 memcached 不会互相通信以共享信息。分布式完全取决于客户端的实现。

        在 Windows 上开启两个 memcached server, 第二个如下开启
            > memcached.exe -p 9999

<?php
    // 本机上有两个 memcached 服务
    $mem = new Memcache;
 
    $mem->addServer('127.0.0.1', 11211);
    $mem->addServer('127.0.0.1', 9999);
 
    // 注意：把 key1 放入到 11211 端口所在服务器还是 9999 所在服务器，由 $mem 对象本身维护
    // 会自动根据分布式算法放入某一个服务器
    if( $mem->set('key1', 'hello', MEMCACHE_COMPRESSED, 300) ){
        echo "Add OK";
    }
 
    if( $mem->set('key2', 'hello2', MEMCACHE_COMPRESSED, 300) ){
        echo "Add OK";
    }
 
    if( $mem->set('key3', 'hello3', MEMCACHE_COMPRESSED, 300) ){
        echo "Add OK";
    }

<?php
 
    // 如何从多个 memcached 服务器中取出相应的 key
 
    $mem = new Memcache;
 
    $mem->addServer('127.0.0.1', 11211);
    $mem->addServer('127.0.0.1', 9999);
 
    $val = $mem->get('key1');
 
    echo '程序中取出分布的值'.$val;



总结

    1. memcached 服务的数据不是同步的，数据是分布的。

    2. 把什么数据放入到哪个 memcached 是由客户端的 memcached 对象决定

    3. 当执行 addServer 时，并不是立即去连接 memcached 服务，而是通过计算，hash 后才去决定连接哪个 memcached 服务，因此当你大量加入服务器到连接池，并没有多余的开销。
