C10K Problem

    The C10k problem is the problem of optimising network sockets to handle a large number of clients at the same time.

    The name C10k is a numeronym for concurrently handling ten thousand connections.

    Note that concurrent connections are not the same as requests per second.  A system capable of handling high number of concurrent connections does not necessarily have to be a fast system, only one where each request will deterministically return a response within a (not necessarily fixed) finite amount of time.
