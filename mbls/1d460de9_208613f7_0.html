四、数据字段属性
   1. unsigned 可以让空间增加一倍  -128-127   0-255
       只能用在数值型字段
       插入负值则会按最小极限0的值插入
   2. zerofill
       只能用在数值型字段， 前导0
       该字段自动应用UNSIGNED
   3. AUTO_INCREMENT
       只能是整数， 数据每增加一条就会自动增1， 字段的值是不充许重复（因而通常为唯一索引）
       NULL，0，留空，值是这3种情况下都会自动加1
       必须为KEY，如后面加上 primary key
       从已经插入过的最大值开始增加，留空的值不会补上
       每个表都最好有一个ID字段，设置为自动增涨， auto_increment
   4. NULL 和 NOT NULL
       默认是空
       是一个值，NULL 值
       将来将这个表的数据转为PHP程序的数据时， 整数列有NULL  转成0吗  字符串NULL  ' '  double 0.00        //不确定！！
               建议： 在创建表时每个字段都不要插入NULL
               加上限定：not null
               若值都是not null, 但指定只插入一部分，如下，则会插入0.
                   create table t4(id int not null, name varchar(30) not null, price double not null);
                   insert into t4(name) values('aaa');
               下面则会报错
                   insert into 54 values(null, null, null);

五、创建索引
   1. 主键索引
       主要作用是确定数据库表里一条特定数据记录的位置
       最好为每一张数据表定义一个主键
       一个表只能指定一个主键
       主键的值不能为空
   2. 唯一索引
       都可以防止创建重复的值
       每个一表都可以有多个唯一索引
       使用关键字：unique
    3. 常规索引
       最重要的技术
             //提升数据库的性能
           //索引优化首先考虑常规索引
               有索引后可以按索引顺序
       可以提高查找的速度， 但会减慢数据列上插入，删除， 修改的速度
                       和表一样是独立的数据对象
       可以单独使用
               create index ind1 on users(name, age);
                       //在users表的 name 和 age上创建名为ind1的索引
               drop index ind1 on users
                       //删除索引
       也可在创建表时创建
               create table carts(
                   id int not null,
                   uid int not null,
                   sid int not null,
                   number int not null,
                   primary key(id),
                   key cuid(uid),
                   index csid(sid));
                       //index key 是同义词，多列都可以一起或单个分别指定索引
                       //如，也可以合在一起 index cid(sid, udi);
                       //删除时分别需1次和2次删除
   4. 全文索引
       不使用匹配的条件下去搜索单词或短语，将所有单词生成一个清单
       fulltext类型索引，只能在 MyISAM 表类型使用， 只有在varchar char text文本字符串上使用
       可以1个或多个数据列使用
               create table books(
                       id int,
                       bookname varchar(30),
                       price double,
                       detail text not null,
                       fulltext(detail, bookname),
                       index ind(price),
                       primary key(id) );
       非全文索引，效率低
               select * from books where bookname like "%php%";
       全文索引
               select bookname, price from books where MATCH(detail) AGAINST('php');
               select match(detail) against("php") from books;                        
六、数据表类型及存储位置
       MySQL和大多数数据库不同，MySQL 有一个存储引擎概念
       针对不同的存储需求，可以选择最优的存储引擎。
       存储引擎：也称数据表类型
               show engines;        //查看支持的存储引擎
               show variables like 'table_type';        // 查看默认的存储引擎
       只学其中的MyISAM 和 InnoDB两个
       创建表的时间指定引擎，不用默认的
               create table () type InnoDB;        //or,
               create table() engine InnoDB;
       MyISAM表类型是默认的，选择MyISAM还是选择InnoDB
       在同一个MySQL库中可以（创建表时）指定不同的表类型
       MyISAM表类型
               配置文件中默认指定此表类型
               成熟稳定，易于管理
               采用表格锁定的机制，用来优化多个并发的操作
               代价，创建完表之后经常要使用
                       OPTIMIZE TABLE 来恢复更新机制所浪费的空间
                       即增删改查出现碎片，进行整理
               强调快速读取操作
               缺点：有一些功能不支持，如不支持外键。（较少使用外键了）
               如果两个表类型都支持相同的功能，可以选择此表类型
       InnoDB 表类型
               可以看成上MyISAM表类型的更新换代的产品
               提供事务处理的：提交、回滚、崩溃恢复等
               支持外键的机制
               若需更高的安全性，很多用户同时修改数据表，考虑InnoDB
               空间占用量比MyISAM大的多，读写操作速度比MyISAM慢
               也不支持全文索引等
       比较：
               功能                        MyISAM                        InnoDB
               --------------------------------------------------------------------------------
               事务处理                不支持                                支持
               数据行锁定                不支持                                支持
               外键约束                不支持                                支持                        
               表空间占用                相对小                                相对大，最大2倍
               全文索引                支持                                不支持
       MySQL表存储
               每一个表都以文件的形式存储在磁盘上
               MyISAM:
                       每个表至少有一个存储表结构的文件: *.frm
                       存储内容，数据:MYD
                       存储索引的MYI
               InnoDB: 只有 frm类型文件

四、数据字段属性
   1. unsigned 可以让空间增加一倍  -128-127   0-255
       只能用在数值型字段
       插入负值则会按最小极限0的值插入
   2. zerofill
       只能用在数值型字段， 前导0
       该字段自动应用UNSIGNED
   3. AUTO_INCREMENT
       只能是整数， 数据每增加一条就会自动增1， 字段的值是不充许重复（因而通常为唯一索引）
       NULL，0，留空，值是这3种情况下都会自动加1
       必须为KEY，如后面加上 primary key
       从已经插入过的最大值开始增加，留空的值不会补上
       每个表都最好有一个ID字段，设置为自动增涨， auto_increment
   4. NULL 和 NOT NULL
       默认是空
       是一个值，NULL 值
       将来将这个表的数据转为PHP程序的数据时， 整数列有NULL  转成0吗  字符串NULL  ' '  double 0.00        //不确定！！
               建议： 在创建表时每个字段都不要插入NULL
               加上限定：not null
               若值都是not null, 但指定只插入一部分，如下，则会插入0.
                   create table t4(id int not null, name varchar(30) not null, price double not null);
                   insert into t4(name) values('aaa');
               下面则会报错
                   insert into 54 values(null, null, null);

索引：

       多占用磁盘空间
