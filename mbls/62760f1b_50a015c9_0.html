检测页面是否可以使用

    jQuery 的 .ready() 方法用于检测页面是否已经准备好让你的代码进行操作。    

    $(document) 创建一个 jQuery 对象来表示当前页面

        // 页面创建好后 function(){...} 中的代码就会执行
        $(document).ready( function(){
            // code goes here
        });


    在上述方法中，脚本在页面中的任意位置，甚至另一文件中时也会正常的工作。


    简写

        $(function(){
            // your script goes here
        });



load 事件

    jQuery .load() 方法，在 load 事件后触发 

    已被 .on() 方法取代

    在页面及其所有资源（图片, css, 脚本)都加载完成后触发



.ready() 方法

    会检测浏览器是否支持 DOMContentLoaded 事件（此事件在 DOM 加载完成后立刻触发，不等待其他资源加载）

    若支持 jQuery 会创建一个事件监听器来响应此事件，否则（在旧浏览器中）会等待 load 事件来触发



让脚本紧跟在 </body> 标签之前

    则在脚本运行之前，HTML 就已经被加载到 DOM 中。
