消除冲突

    ## 从 master 的历史版本派生的分支 Fix-B，与当前 master 分支合并时，产生冲突
    ## 系统提示， README.md 文件发生了冲突(Conflict)，不解决冲突就无法完成合并
    $ git merge --no-ff fix-B

    ## 打开冲突文件 README.md，以解决冲突
    $ vi README.md

    
    ## 冲突文件 README.md 内容如下
    ## ========= 前的是当前 master 分支的内容（HEAD)
    ## ========= 下是 fix-B 分支的内容
    <<<<<<< HEAD
    hi
    =======
    fix-B something
    >>>>>>> fix-B


    ## 修改好冲突的 README.md 文件内容之后
    ## 执行 git add & git commit 提交解决后的结果，即解决了冲突
    $ git add README.md
    $ git commit -m "Fix conflict"
