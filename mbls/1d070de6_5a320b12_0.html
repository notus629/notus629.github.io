字符串类型 - 6 组 * N

    CHAR/VARCAHR, BINARY/VARBINARY, TEXT, BLOB, ENUM, SET


TypeDescription
[NATIONAL] CHAR[(M)]
[CHARACTER SET charset_name]
[COLLATE collation_name]		Fixed-length string, right-padded with spaces to the specified length when stored. 
M represents the column length in characters.
The range of M is 0 to 255. If M is omitted, the length is 1
CHAR / CHARACTERCHAR(0): old applications that depend on the existence, but not use
NCHAR / NATIONAL CHAR The standard SQL way to define that a CHAR column should use some predefined character set.
MySQL uses utf8
BINARY / CHAR BYTE This is a compatibility feature.
[NATIONAL] VARCHAR(M)
[CHARACTER SET charset_name]
[COLLATE collation_name]		A variable-length string.
M represents the maximum column length in characters. The range of M is 0 to 65,535. 
(utf8: 3 bytes per character. M Maximum: 21844 characters)
VARCHAR / CHARACTER VARYING
NVARCHAR / NATIONAL VARCHARThe standard SQL way to define that a VARCHAR column should use some predefined character set. MySQL uses utf8
BINARY(M)Stores binary byte strings rather than nonbinary character strings.
M represents the column length in bytes.
VARBINARY(M)Stores binary byte strings
M represents the maximum column length in bytes
TEXT[(M)]
[CHARACTER SET charset_name]
[COLLATE collation_name]		A TEXT column with a maximum length of 65,535 (2^16 − 1) characters.
Stored using a 2-byte length prefix that indicates the number of bytes in the value.
If M is specified, MySQL creates the column as the smallest TEXT type large enough to hold values M characters long.

LONGTEXT
[CHARACTER SET charset_name] [COLLATE collation_name]		Maximum length of 4,294,967,295 or 4GB (2^32 − 1) characters.
Effective maximum length depends on the configured maximum packet size in the client/server protocol and available memory.
Stored using a 4-byte length prefix that indicates the number of bytes in the value.
MEDIUMTEXT
[CHARACTER SET charset_name]
[COLLATE collation_name]		Maximum length of 16,777,215 (2^24 − 1) characters.
Stored using a 3-byte length prefix that indicates the number of bytes in the value.
TINYTEXT
[CHARACTER SET charset_name]
[COLLATE collation_name]		A TEXT column with a maximum length of 255 (2^8 − 1) characters.
The effective maximum length is less if the value contains multibyte characters.
Stored using a 1-byte length prefix that indicates the number of bytes in the value.
BLOB[(M)]A BLOB column with a maximum length of 65,535 (2^16 − 1) bytes.
Stored using a 2-byte length prefix that indicates the number of bytes in the value.
If M is specified, MySQL creates the column as the smallest BLOB type large enough to hold values M bytes long.
LONGBLOBMaximum length of 4,294,967,295 or 4GB (2^32 − 1) bytes.
Effective maximum length depends on the configured maximum packet size in the client/server protocol and available memory.
Stored using a 4-byte length prefix that indicates the number of bytes in the value.
MEDIUMBLOBMaximum length of 16,777,215 (2^24 − 1) bytes.
Stored using a 3-byte length prefix that indicates the number of bytes in the value. 
TINYBLOBA BLOB column with a maximum length of 255 (2^8 − 1) bytes.
Each TINYBLOB value is stored using a 1-byte length prefix that indicates the number of bytes in the value.
ENUM('value1','value2',...)
[CHARACTER SET charset_name] [COLLATE collation_name]		An enumeration.
A string object that can have only one value, chosen from the list of values, NULL or the special '' error value.
ENUM values are represented internally as integers.
Maximum of 65,535 distinct elements.
SET('value1','value2',...)
[CHARACTER SET charset_name] [COLLATE collation_name]		A set.
A string object that can have zero or more values, chosen from the list
SET values are represented internally as integers.
Maximum of 64 distinct members.



Attribute

    CHARACTER SET / CHARSET    specifies the character set


    COLLATE                    specifies a collation for the character set

        CREATE TABLE t
        (
            c1 VARCHAR(20) CHARACTER SET utf8,
            c2 TEXT CHARACTER SET latin1 COLLATE latin1_general_cs
        );



CHARACTER SET

    binary    

        causes the column to be created as the corresponding binary string data type

        CHAR becomes BINARY, VARCHAR becomes VARBINARY, and TEXT becomes BLOB


    ASCII

        aka CHARACTER SET latin1


    UNICODE

        aka CHARACTER SET ucs2
