5. Using cp and the differences between cp –a/-r/-p?

A. 
        cp synposis

                cp [选项] file1 file2       #file1拷贝为file2（file1,file2都非目录）

                cp [选项] file dir           #file 拷贝到dir目录下（file不包含目录）

                cp [OPTION] ... –t DIRECTORY SOURCE       #参见-t 选项

        选项

                -a, --archive 
                same as -dR --preserve=all

                -d same as --no-deference –preserve=links

                -P --no-derefernece
                never follow symbolic links in SOURCE

                -p same as –preserve=mode, ownership, timestamps

                --perserve[=ATTR_LIST]
                preserve the specified attributes (default: mode, ownership, time-stamps), if possible additional attributes: context, links, xattr, all

                --parents
                use full source file name under DIRECTORY
                (复制时在目标文件中创建源文件路径中的所有目录，当然除根目录/外，也可以说有根，因为根/就是路径分隔符.并且源文件列出了绝对路径，Leo注)

                 -R, -r, -recursive
                copy directories recursively

                -t, --target-directory=DIRECTORY
                copy all SOURCE arguments into DIRECTORY


        对于cp命令，可以直接 cp file1 file2，把一个文件重命名（原文件不变），但目录不行（目录不能直接cp dir1 dir2）来重命名。cp –a dir1 dir2 可以达到这个目的（即复制出一个同样的目录，非空目录也可以）cp –r dir1 dir2 可以达到同样的目的。如果两种情况下的dir2已经存在，则是将dir1复制到dir2中去。


        比较：-a选项是包含了-r，-a保留了所有属性，而-p只是保留mode, ownership, timestamps等属性

9. 在VirtualBox 4.3.30中安装CenOS-5.10后，即使利用光盘安装了VirtualBox增强组件，仍然无法调整分辨率?

原因：安装没有完全成功。即：Bulding the VirtualBox Guest Additions Kernel modules failed

解决方案：正确安装CD里的VBoxLinuxAdditions-x86.run 或 VBoxLinuxAdditions-amd64.run。

步骤：
a. 挂载光盘
b. 安装编译内核的相关组件。
yum install kernel-devel gcc
c. 在安装完成后，做一个连接
ln -s /usr/src/kernels/2.6.18-92.1.18.el5-i686 /usr/src/linux
其中“2.6.18-92.1.18.el5-i686”是内核的版本号，不同版本不一样，需要根据自己的系统内核填入。如何查看虚拟机上的Linux的平台、内核版本号：
cat /proc/version or cat /etc/issue or uname -a
d. 安装VirtualBox客户端增强功能工具，执行以下命令来安装
sh ./VBoxLinuxAdditions.run
e. 安装成功后重启系统。

11. Linux文件权限

特殊权限
其实文件与目录设置不止这些，还有所谓的特殊权限。由于特殊权限会拥有一些“特权”，因而用户若无特殊需求，不应该启用这些权限，避免安全方面出现严重漏洞，造成黑客入侵，甚至摧毁系统!!!
s或S（SUID,Set UID）：可执行的文件搭配这个权限，便能得到特权，任意存取该文件的所有者能使用的全部系统资源。请注意具备SUID权限的文件，黑客经常利用这种权限，以SUID配上root帐号拥有者，无声无息地在系统中开扇后门，供日后进出使用。
s或S（SGID，Set GID）：设置在文件上面，其效果与SUID相同，只不过将文件所有者换成用户组，该文件就可以任意存取整个用户组所能使用的系统资源。
T或T（Sticky）：/tmp和 /var/tmp目录供所有用户暂时存取文件，亦即每位用户皆拥有完整的权限进入该目录，去浏览、删除和移动文件。
因为SUID、SGID、Sticky占用x的位置来表示，所以在表示上会有大小写之分。加入同时开启执行权限和SUID、SGID、Sticky，则权限表示字符是小写的：
-rwsr-sr-t 1 root root 4096 6月 23 08：17 conf
如果关闭执行权限，则表示字符会变成大写：
-rwSr-Sr-T 1 root root 4096 6月 23 08：17 conf



12. chmod 中，大写X权限的用途
[ref]
man中解释，x就是给文件加上执行权限；而大写X，是当文件为目录或者某个用户拥有可执行文件的前提下，作为可执行权限用途。
意思是，比如，若原始普通非目录文件任何用户都没有可执行权限，那a+X还是没有可执行权限。若原始普通非目录文件的拥有者是可执行权限，而其它用户没有可执行权限或者该文件就是一个目录，则a+X会使所有用户都拥有可执行权限



13. Linux version numbers explained
a. Usually four numbers, sometimes supplemented by several additional characters.
First number denotes the kernel version, seldom changed.
Second number denotes the major revision of kernel version. Even numbers indicated a stable release. Odd numbers, have historically represented development releases. They were for testing new features and device drivers until they became sufficiently stable to be included in a stable release. However, this has changed starting with the Linux 2.6.x series, and new feature development now takes place in the same revision number.
Third number indicates the minor revision of the kernel. It is only changed when new features or new drivers are added.
Fourth number represents corrections, such as security patches and bug (i.e., error) fixes.
Sometimes the four numbers will be followed by several letters, such as rc1, ac, ck and mm. The letters rc (followed by a number) refer to a release candidate and thus indicate a non-official release. Other letters usually indicate the person responsible for that release, such as Alan Cox, Con Kolivas and Andrew Morton.
b. What does “stable/EOL” and “longterm” mean?
stable/EOL: EOL(end of life), no more bugfix revisions.
longterm: more bugfixes will be provided
stable(leo add): just a stable version for use, most people should download this version for current use.

15. Linux 动态库(so, shared object)与静态库
? 存放位置。绝大多数存放在/lib、/usr/lib/,完整的动态库存放路径列表可通过/etc/ld.so.conf文件配置（若修改了配置，需要用/sbin/ldconfig命令更新缓存）
? 动态库搜索路径并不包括当前文件夹，所以即使可执行文件和其所需的so文件在同一文件夹，也会出现找不到so的问题
? 版本：动态库的版本总是个问题，如果编译时链接的库和执行时提供的不一样，难免会导致程序的执行发生诡异的错误。在Linux中：
? 每个so文件有一个文件名，如libABC.so.x.y.z，这里ABC是库名称，x.y.z是文件的版本号
? 第一位x表示了兼容性，x不一样的so文件是不能兼容的。
? 第二位y的变化表示可能引入了新的特性（Feature），但总的来讲还是兼容的。
? 第三位z的变化一般表示仅是修正了Bug。
? ldd “程序名字” #查看程序调用了哪些动态链接库
? ldconfig #加载动态链接库
? ldconfig –v #查看哪些动态链接库已经被加载了
? gcc –static hello1.c #阻止使用静态库
? gcc hello1.c #gcc默认使用动态库
? 制作静态库方法
? 把代码为目标文件形式（不链接）
gcc -c liberr.c -o liberr.o
? 使用工具ar创建一个存档文件(静态库)
ar rcs liberr.a liberr.o
? 编译程序时把程序和liberr.a链接起来：
gcc errtest.c -o errtest -static -L./ -lerr
? 制作动态库的方法
首先编译目标文件
 gcc -fPIC -c liberr.c -o liberr.o

20. ldd命令
主要用于查看程式运行所需的共享库
输出：libc.so.6 => /lib/i686/libc.so.6 (0x42000000)
‘ => ’ 前面是为动态链接程序所需的动态链接库的名字
‘ => ’后面的是运行时系统实际调用的动态链接库的名字（如果需要的动态链接库在系统中不存在时，‘ => ’后面将显示"not found"）。
‘ (0xXXXXXXX) ’用括号括住的数字属于虚拟的执行地址。
