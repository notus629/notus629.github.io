AccessToken 与 RefreshToken 生命周期

    AccessToken 具有较长的生命周期，10天半个月甚至更长，短的也有几小时

    可以在 User Authorization URL 中指定参数

        ... &need_refresh_token=true&...

        这样在返回的数据中会带有新的 AccessToken, 各平台 need_refresh_token 参数名字不尽相同

    

注记：

    1. RefreshToken 不常用，通常一些定时任务，比如用户不登录通过站点来定时发说说。此处暗指登录时会自动刷新 AccessToken?

    2. 课程中提到 RefreshToken 也有生命周期（更久，可以半年），上面写的RefreshToken不是一个布尔值吗？难道和 need_refresh_token 不是一回事，亦或还有一个特定的 RefreshToken 值，过期后必须用户登录来获取，而不能再刷新 AccessToken?
