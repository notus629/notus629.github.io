git diff - 查看更改前后的差别

    git diff 命令，查看工作树、暂存区、最新提交之间的差别。


    若未进行 git add 和 git commit, 即暂存区和最新提交均为空，则 git diff 无输出显示

    若工作区更改已全部提交，则 git diff 无输出显示



查看工作树和暂存区的差别

    ## 仓库中已有提交，工作树有最新修改，但尚未添加到暂存区
    ## 暂存区无内容，则 git diff 显示工作树和最新提交状态之间的差别
    $ git diff

    输出如下：
        diff --git a/README.md b/README.md            ## --git a/README.md, 仓库中的文件； b/README.md 工作树中的文件
        index 4d1a8d6..f1b00ba 100644                 
        --- a/README.md                               ## --- a/README.md, 仓库中文件，前面用 - 前缀, 删除的内容
        +++ b/README.md                               ## +++ b/README.md, 工作树中的文件，前面用 + 前缀，添加的内容
        @@ -1 +1 @@                                   ## -1 仓库中的第一行，与 +1 工作树中的第一行比较
        -# Git tutorial                               ## -# Git tutorial, 仓库中第一行内容为: # Git tutorial
        +# Git tutorial, changed by leo,,,,,,haha     ## +# Git tutorial, changed by leo,,,,,,haha, 工作树中第一行内容为：...

    
    ## 若此时用 git add 将 README.md 添加到暂存区
    ## 则工作树与暂存区没有差别，git diff 无显示
    $ git add README.md
    $ git diff    

    ## 查看与最新提交的差别
    ## 可养成习惯，每次提交之前 git diff HEAD, 查看本次提交与上次提交的差别
    ## HEAD 指向当前分支中最新一次提交的指针
    $ git diff HEAD
