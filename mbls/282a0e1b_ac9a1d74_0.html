遇到问题

    若新在 git 上建一个仓库，本地使用使用下面两行命令。会遇到 git 拒绝推送，因为有冲突。

    ## 添加远程仓库的本地标识符
    $ git remote add origin git@github.com:obpxnfxm/phppractice.git    

    ## 将本地当前分支推送到 orign 仓库的 master 分支
    $ git push -u origin master                                         



解决思路

    方案一：在 Git 上新建一个 repo（包含了License, .ingnore 或 README.md），然后 clone 到本地，再在本地更改代码，提交后上传到 GitHub.


    方案二：若在 GitHub 上新建 repo 时，没有 License, .ingnore, README.md, 直接 push 也不应该会有问题。


    其它方案：网上搜索，先应将远程 git 上的仓库 fetch 到本地，再与本地合并。或者直接用 git pull, 即相当于 fetch 之后再合并。此时会遇到 "refusing to merge unrelated histories" 的错误。

    再 git pull 后面加上 --allow-unrelated-histories, 解决问题，命令如下。

        $ git pull origin master --allow-unrelated-histories



步骤应该如下：

    1. 本地 git init

    2. git remote add     

    3. git pull

    4. git add & commit 

    5. git push


    下次尝试直接 pull，而省略第 1 步
