Redefine tab as 4 spaces

$vim ~/.vimrc

set tabstop=4                " The width of a TAB is set to 4. Still it is a \t. It is just that Vim will interpret it to be having a width of 4.
set softtabstop=4        " make the tab key (in insert mode) insert a combination of spaces (and possibly tabs) to simulate tab stops at this width.
set expandtab              " Expand TABs to spaces
set shiftwidth=4            " Indents will have a width of 4

保存文件并退出
:x
:wq

这两个命令实际上并不完全等价，当文件被修改时两个命令时相同的。但如果未被修改，使用 :x 不会更改文件的修改时间，而使用 :wq 会改变文件的修改时间。


自动补全

Vim 默认有自动补全的功能。的确这个功能是很基本的，并且可以通过插件来增强，但它也很有帮助。方法很简单。Vim 尝试通过已经输入的单词来预测单词的结尾。比如当你在同一个文件中第二次输入 "compiler" 时，仅仅输入 "com" 然后保持在插入模式，按 Ctrl+n 键就可以看到 Vim 为你补全了单词。很简单，但也很有用。

Vim打开utf8编码文件乱码的解决方法 (2015-04-07 21:06:07)
http://blog.sina.com.cn/s/blog_74a7e56e0102vh44.html
Vim 有四个跟字符编码方式有关的选项: encoding、fileencoding、fileencodings、termencoding.
其各自意义如下:
encoding: Vim 内部使用的字符编码方式，包括 Vim 的 buffer (缓冲区)、菜单文本、消息文本等。
fileencoding: Vim 中当前编辑的文件的字符编码方式，Vim 保存文件时也会将文件保存为这种字符编码方式 (不管是否新文件都如此)。
fileencodings: Vim 启动时会按照它所列出的字符编码方式逐一探测即将打开的文件的字符编码方式，并且将 fileencoding 设置为最终探测到的字符编码方式。因此最好将 Unicode 编码方式放到这个列表的最前面，将拉丁语系编码方式 latin1 放到最后面。
termencoding: Vim 所工作的终端的字符编码方式。该选项在 Windows 下对我们常用的 GUI 模式的 gVim 无效，而对 Console 模式的 Vim 而言就是 Windows 控制台的代码页，并且通常我们不需要改变它。用英文菜单和提示最好，可以免去下面对菜单和提示信息（B，C部分）的设置如果用英文菜单和提示在安装gvim的时候，将支持本地语言的选项去掉。
解决vim文件乱码:
(1) 设置文件的编码形式:
set encoding=utf-8
set termencoding=utf-8
set fileencoding=utf-8
set fileencodings=ucs-bom,utf-8,chinese
(2) Vim菜单乱码:
同样在 _vimrc文件里以上的中文设置后加上下列命令，
source $VIMRUNTIME/delmenu.vim
source $VIMRUNTIME/menu.vim
(3) Vim提示信息乱码:
language messages zh_CN.utf-8



Remove swap and backup files from your working directory(TBC...)
ref: http://vim.wikia.com/wiki/Remove_swap_and_backup_files_from_your_working_directory
               set backupdir=./.backup,.,/tmp
               set directory=.,./.backup,/tmp
       This way, if you want your backups to be neatly grouped, just create a directory called '.backup' in your working directory. Vim will stash backups there.
       The 'directory' option controls where swap files go. If your working directory is not writable, Vim will put the swap file in one of the specified places.
vim swap file:
       if your file name is page.inc,  then vim auto generated swap file is .page.inc.swap
