PostCSS 

    PostCSS 是使用 JS 插件来转换 CSS 的工具，支持变量，混入，未来 CSS 语法，内联图像等等

    PostCSS 已经被许多大公司使用：谷歌，推特，阿里巴巴和 Shopify。 Autoprefixer PostCSS 插件是最流行的 CSS 处理器插件之一。

    PostCSS 可以作为预处理器使用，类似：Sass, Less 和 Stylus。但是 PostCSS 是模块化的工具，比之前那些快3-30 倍，而且功能更强大。

    PostCSS是CSS变成JavaScript的数据，使它变成可操作。PostCSS是基于JavaScript插件，然后执行代码操作。PostCSS自身并不会改变CSS，它只是一种插件，为执行任何的转变铺平道路。    



sass & less

    Sass 和 LESS 是一样的，只是 CSS 预处理器中的一种而以.

    Sass 在市面上有一些成熟的框架，比如说 Compass，而且有很多框架也在使用 Sass，比如说 Foundation. 就国外讨论的热度来说，Sass 绝对优于 LESS ([1], 2013)

    Bootstrap ported from Less to Sass for easy inclusion in Rails, Compass, or Sass-only projects.[2]

    Sass的功能比Less强大,基本可以说是一种真正的编程语言了，Less则相对清晰明了,易于上手,对编译环境要求比较宽松。考虑到编译Sass要安装Ruby,而Ruby官网在国内访问不了,个人在实际开发中更倾向于选择Less。[4]

    Bootstrap 4 已经迁移到了 Sass [6][7]



Scss [5]

    Sass has two syntaxes. The new main syntax (as of Sass 3) is known as "SCSS" (for "Sassy CSS"), and is a superset of CSS's syntax. This means that every valid CSS stylesheet is valid SCSS as well. SCSS files use the extension .scss.

    

Stylus ([1] 2016-5)

    目前（2017-5-4）用的人还较少。

    LESS 用的稍微深入一点你就会发觉它 hold 不住了。自定义函数不支持，运算符也少的可怜，还用坑爹的 css 关键字 @ 符声明变量。

    SASS 的问题在于…嗯，安装好麻烦，用过 node-sass 的人都懂，都是泪 。经历了这些你就会很希望有一个功能全面使用方便的速度快的编译工具，所以stlyus 出现了。
