gitpull & pull request

    git pull

        是把 git 仓库（如码云）的内容拉到本地，去本地代码合并。即更新


    pull request 

        你 clone 一份别人的代码，但是你发现别人代码写的有问题，你想改了以后提交，但是你又没有权限……

        这时候，你建一个新分支，修改，commit 后，发起 pull request。代码的原作者就会收到一个 pull request 请求。他可以去把你的代码 clone 下来，如果他检查没有问题了，可以合并到他的原始代码里。



pull request [1]

    拉请求，请求作者将你的补丁拉过去，合并到原来代码中。

    你将某个仓库的代码 fork 之后，就 fork 到了你的仓库，然后进行了修改。

    再请求对方 pull 这个修改后的 repo 到对方主分支进行合并，叫 pull request

    如果你本来有权限创建一个分支，则叫 merge request
