git reset - 回溯历史版本

    git reset --hard    让仓库的 HEAD， 暂存区， 当前工作树回溯到指定状态   



回溯到指定的历史版本

    ## hash_code: 指定目标点的 hash 值（4 位以上即可）
    $ git reset --hard hash_code 



在该历史版本创建特性分支 fix-B

    ## 创建分支 fix-B
    $ git checkout -b fix-B
    ## 修改 README.md 后提交
    $ git add README.md
    $ git commit -m "Fix B"



再推进历史 - 推进至回溯前的最新提交状态 （本例中是 feature-A 分支合并后的状态）

    ## 查看当前仓库执行过的操作的日志
    ## 只要不进行 Git 的 GC （Garbage Collection 垃圾回收），就可以通过日志随意调取近期的历史状态
    ## 从日志中找到目标点（回溯前的状态点）的 hash code
    $ git reflog
        
    ## 将 HEAD, 暂存区，当前工作树恢复到回溯前状态点
    $ git checkout master
    $ git reset --hard hash_code
