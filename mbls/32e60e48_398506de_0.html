设置 SSH Key



准备知识

    GitHub 上连接已有仓库时的认证，可采用 SSH 的公开密钥认证方式进行

        这样不用在每次连接到仓库时输入 GitHub 的用户名和密码，更方便，也更安全


    须创建公开密钥认证所需 SSH Key 并将其添加至 GitHub

    
    此后每次连接至 GitHub 时，就会使用用户本地的私钥，而 GitHub 用该用户已添加的公钥来验证用户的身份



创建 SSH Key

    ## 提示输入保存 SSH Key 的文件对 （key pair) 的名称
    ## 默认路径和名称：/Users/your_user_directory/.ssh/id_rsa
    ## 再输入此 key 对应的密码，再重复此密码
    ## 执行结果：默认在生成 id_rsa, id_rsa.pub 两个文件（即 key pair)
    ## 成功后会在命令行显示： key fingerprint 及 key's randomart image
    $ ssh-keygen -t rsa -C "geneleo@qq.com"

    

将公开密钥(id_rsa.pub) 添加至 GitHub

    GitHub -> Settings -> SSH and GPG keys -> SSH keys -> New SSH key

        Title: 输入适当的密钥名称

        Key: 将本地生成的 id_rsa.pub 公开密钥的内容复制后，粘贴到此处


    完成后即可利用手中私人密钥与 GitHub 进行认证和通信



利用 SSH Key 登录 GitHub

    ## 提示 Enter passphrase for key '/c/Users/Gene/.ssh/id_rsa'
    ## 输入在生成 ssh key pair 时设置的密码即可
    ## 这里仅仅是测试能否利用 SSH Key 连接上 GitHub, 实际 GitHub 不支持此种交互方式
    $ ssh -T git@github
