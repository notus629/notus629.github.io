使用 composer 下载一个依赖库的两个步骤

    a. 声明依赖：把相关库的信息添加到 composer.json 配置文件中（在当前项目根目录下）

        ## 把 monolog/monolog 1.21.0 加载到 composer.json 配置中, composer.json 文件内容如下
        {
            ...
    
            "require" : {
                "monolog/monolog":"1.21.*"
            }
        }


    b. 配置好之后，安装依赖 $ composer install



也可以用 composer require 命令来声明依赖

    将自动添加依赖到 composer.json 配置文件中，

    并自动开始下载

    ## 示例
    $ composer require symfony/http-foundation



手工删除依赖

    a. 删除 composer.json 中的依赖声明

    b. 执行 $ composer update 即可



在项目中使用

    需要的 composer 包下载好之后，就会在当前目录下生成一个 vendor 目录，并将下载的包放入其中，并且生成一个 autoload.php 文件

    在项目只需要 require 'vendor/autoload.php'; 即可自动加载需要用的组件了, 即可如下使用：

        $myLog = new \monolog\Logger('wangshuo');
