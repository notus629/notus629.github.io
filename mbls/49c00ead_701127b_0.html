git push - 推送至远程仓库



推送至 master 分支

    ## 假定当前在 master 分支下，当前分支的内容被推送给远程仓库 origin 的 master 分支 
    ## -u， 在推送的同时，将 origin 仓库的 master 分支设置为本地仓库当前分支的 upstream（上游），
    ## 这样，在 git pull 从远程仓库获取内容时，本地仓库的这个分支就可以直接从 origin 的 master 分支获取
    $ git push -u origin master



推送至 master 以外的分支

    ## 在本地仓库创建了 feature-D 分支，并切换至该分支
    $ git checkout -b feature-D

    ## 将本地的 feature-D 分支，push 至远程仓库，并保持分支名称不变
    ## （即将本地当前分支内容 push 至在远程仓库新创建的同名分支, GitHub 上会自动创建该分支）
    $ git push -u origin feature-D



注：

    设置好本地某分支的上游分支后，再次推送时，只需 $ git push 即可
