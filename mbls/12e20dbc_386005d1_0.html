What's different between UTF-8 and UTF-8 BOM?

    1. UTF-8 BOM（byte order mark），比 UTF-8 多了 3 个字节(EF BB BF)，是为了标明字符编码的字节序.

    2. BOM 并无必要，标准中不推荐，主要由微软的 Windows 使用，UTF-8 才是标准。



详细解释

    The UTF-8 BOM is a sequence of bytes (EF BB BF) that allows the reader to identify a file as being encoded in UTF-8.

    Normally, the BOM is used to signal the endianness of an encoding, but since endianness is irrelevant to UTF-8, the BOM is unnecessary. 

    BOM（byte order mark）是为 UTF-16 和 UTF-32 准备的，用于标记字节序（byte order）。微软在 UTF-8 中使用 BOM 是因为这样可以把 UTF-8 和 ASCII 等编码明确区分开，但这样的文件在 Windows 之外的操作系统里会带来问题。

    According to the Unicode standard, the BOM for UTF-8 files is not recommended. 因而 UTF-8 不需要 BOM，尽管 Unicode 标准允许在 UTF-8 中使用 BOM。

    所以不含 BOM 的 UTF-8 才是标准形式，在 UTF-8 文件中放置 BOM 主要是微软的习惯（顺便提一下：把带有 BOM 的小端序 UTF-16 称作「Unicode」而又不详细说明，这也是微软的习惯）。
