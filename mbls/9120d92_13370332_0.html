SSH

    SSH， 即 Secure SHell，最初用来代替 telnet，通过 port forward, 可以让其他协议通过 SSH 的隧道而起到加密的效果。

    还可以为 FTP、POP 甚至 PPP 提供一个安全的通道。



OpenSSH

    因受版权和加密算法的限制，大家转而使用 OpenSSH, 是 SSH 的替代软件包，开源软件。

    sshd 命令：是openssh软件套件中的服务器守护进程。



SSL

    SSL 在服务器和客户机两端可以同时被支持，目前已成为互联网上保密通讯的工业标准，现行浏览器普遍将HTTP和SSL相结合，从而实现安全通信。

    SSL与应用层协议独立无关，高层应用协议（如：HTTP， FTP、Telnet 等）能透明的建立于 SSL 协议之上。

    TLS 是 SSL 的高级版本，IETF将其进行了标准化，并改名为 TLS。



SSH 与 SSL 区别

    SSL 和 TLS 被高数加强 Web 安全传输 （HTTP/HTTPS）的协议，（事实上还有 SMTP/NNTP等）。

    而 SSH 更多的被设计为加强 Telnet/FTP 安全的传输协议，默认地，使用 22 端口。

                      SSL              SSH
                +-------------+ +-----------------+
                | Nothing     | |    RFC4254           | Connection multiplexing
                +-------------+ +-----------------+
                | Nothing     | |    RFC4252           | User authentication
                +-------------+ +-----------------+
                | RFC5246     | |    RFC4253           | Encrypted data transport



Linux 开启 ssh 服务 （Ubuntu)

    首先更新源　　
        sudo apt-get update

    安装ssh服务
       sudo apt-get install openssh-server

    检测是否已启动
       ps -e | grep ssh

    看到有ssh字样，说明已启动，如果没有就手动启动
       /etc/init.d/ssh start

    配置ssh-server，配置文件位于/etc/ssh/sshd_config，默认端口为22，为了安全，一般自定义为其他端口，然后重启
       sudo /etc/init.d/ssh restart

    实战步骤

        先看是否启动：$ ps -e | grep ssh       ##如果只有ssh-agent 表示 ssh-server 还没启动, 看到 sshd 则已启动 ssh-server

        没启动需使用：$ /etc/init.d/ssh start 

        若没有安装 ssh, 则需安装：$ sudo apt-get install openssh-server
