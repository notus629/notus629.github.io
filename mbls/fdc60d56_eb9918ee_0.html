需求如下

    1. 实现 host 的目录和 guest 共享，host 编辑的代码可以直接放入此目录，因而在 guest 中也可以读取。

    2. 将 Apache 的 documentroot 设为此共享目录，实现在 host client 上可以直接访问 www web 服务器。

    最终实现，代码编辑修改，及结果呈现全在 host 上完成。



环境：(2017-02-28)

    虚拟机软件： VirtualBox 5.1.14

    Host: Windows 10

    Guest: Windows XP

    Web 环境： wamp5 1.7.4



实现

    需求 1

        通常是在 vbox 设置共享文件夹，在 winxp 中映射驱动器到此共享文件夹即可。


    需求 2

        若此时直接将 apache 的 document 设为此目录（假设为 DocumentRoot ="z:/winxp-www"），重启后会出现 "引用的内存不能 read" 的错误, 其中 z:/ 是映射到 //VBOXSVR/share/ （即 host 的 share 目录）

        此时换一种方式，直接设为 vbox 给定的路径，即在 http.conf 中设置 DocumentRoot =  "//VBOXSVR/share/winxp-www"，则 Apache 可以正常启动，不报错。

        但无论通过 guest 还是 host 访问服务器，都会出现类似 “没有权限访问 / 目录”的问题。此时，在 http.conf 中找到：
        
        <Directory />
        Options FollowSymLinks
        AllowOverride None
        Order deny,allow
        Deny from all
        Satisfy all
        </Directory>

        替换为如下所示即可

        <Directory />
        Options FollowSymLinks
        AllowOverride None
        Order deny,allow
        Allow from all
        Satisfy all
        </Directory>
