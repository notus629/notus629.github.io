XDebug


Configure PHP to use xdebug

    1. add the following line to php.ini 

        zend_extension="/wherever/you/put/it/xdebug.so"
        
        ## for php version earlier than 5.3 and threaded usage of PHP (Apache 2 worker MPM or the ISAPI module)
        zend_extension_ts="/wherever/you/put/it/xdebug.so

    2. restart webserver.



Settings 

    Xdebug's modifed var_dump()

        xdebug.cli_color                 ## Default 0, If this setting is 1, Xdebug will color var_dumps and stack
                                         ## Xtraces output when in CLI mode and when the output is a tty. 
        
        xdebug.overload_var_dump         ## Default : 2, By default Xdebug overloads var_dump() with its own
                                         ## improved version when the html_errors php.ini setting is set to 1 or 2.
        
        xdebug.var_display_max_children  ## Default value: 128, -1, no limitation. not affect remote debugging

        xdebug.var_display_max_data      ## Default 512, -1, no limitation. not affect remote debugging

        xdebug.var_display_max_depth     ## Default 3, -1 or 1023, maximum number. not affect remote debugging

Remote Debugging

    DBGp, the protocol that is being used.  

    Xdebug 2 is bundled with a simple command line client for the DBGp protocol.


    php.ini

        xdebug.remote_enable               ## boolean, default 0, 
        xdebug.remote_host                 ## IP address the debugger should connect to
        xdebug.remote_port                 ## port
        xdebug.remote_connect_back         ## development server is shared with multiple developers
        xdebug.remote_mode                 ## initiate a session when an error situation occurs

        xdebug.remote_handler
        xdebug.remote_log
        redebug.remote_mode


PHPStorm 

    php.ini

        [Xdebug]

            zend_extension=<full_path_to_xdebug_extension>
            xdebug.remote_enable=1
            xdebug.remote_host=<the host where PhpStorm is running (e.g. localhost)>
            xdebug.remote_port=<the port to which Xdebug tries to connect on the host where PhpStorm is running (default 9000)>
