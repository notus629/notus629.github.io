远程连接 MySQL 数据库失败

    Ubuntu 16.04 my.conf 文件位置

        /etc/mysql
        /etc/mysql/my.cnf   to set global options
        ~/.my.cnf           to set user-specific options


    /etc/mysql/mysql.conf.d/mysqld.cnf

        ## 文件中说明可将此文件拷为/etc/my.cnf来设置全局选项
        ## 或将此文件拷贝为~/.my.cnf来设置用户相关的选项


    解决方案：

        在 /etc/mysql/my.cnf 中 设置、bind-address =:: 或 bind-address =0.0.0.0, 重启导致mysql失败

        直接将 /etc/mysql/mysql.conf.d/mysqld.cnf 中 的 bind-address = 127.0.0.1 改为 0.0.0.0 重启， 成功

        远程连接，成功


        
几个参数

    ; 同时连接的个数
    max_connection             
                   

    ; mysql 只能本地访问，注释掉或者改成想要使用的客户端主机IP即可远程 
    bind-address=127.0.0.1    
                    
        0.0.0.0            表示接收所有 IPv4 地址
        ::                 接收所有 IPv4 和 IPv6 地址
        127.0.0.1 或 ::1   只接收本地 IPv4 或 IPv6
