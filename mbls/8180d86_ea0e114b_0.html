授予和取消权限

    命令：GRANT, REVOKE



权限分4个级别

    全局、数据库、表、列



GRANT

    创建用户并赋予他们权限

                GRANT  privileges  [ columns ]
                ON  item
                TO  user_name  [ IDENTIFIED  BY  'password' ]
                [ REQUIRE  ssl_options ]
                [ WITH  [ GRANT  OPTION  |  limit_options ] ]  


        占位符
                privileges: 逗号分开的一组权限

                columns: 可选，对单列或逗号分开的多列指定权限 

                item
                        新权限所应用于的数据库或表
                        *.*  将权限应用于所有数据库，即全局权限
                        *     若没有使用任何特定的数据库，*代表全局权限
                        dbname.*  指定dbname数据库中的所有表
                        dbname.tablename  指定特定数据库中的单个表
                        dbname.tablename 和 columns  共同指定特定的列
 
                user_name
                        用户（被创建并赋予权限的用户）登录时使用的用户名
                        不必同于登录系统时所使用的用户名

                password: 密码

                REQUIRE子句：指明是否必须通过加密套接字连接，或指定其他的SSL选项

                WITH GRANT OPTION选项
                        表示允许指定的用户向别人授予自己所拥有的权限
                
                其它WITH子句
                        MAX_QUERIES_PER_HOUR  n                    // 指定每一个用户每小时执行的查询的数量
                        MAX_UPDATES_PER_HOUR  n                    // 指定每一个用户每小时执行的更新的数量
                        MAX_CONNECTIONS_PER_HOUR  n                // 指定每一个用户每小时执行的连接的数量

 

权限的类型

        MySQL中存在3个基本类型的权限

                适用于赋予一般用户的权限
                适用于赋予管理员的权限
                几个特定的权限


用户的权限 

        Privilege        Applies To                 Description
        SELECT           tables, columns            Allows users to select rows (records) from tables.
        INSERT           tables, columns            Allows users to insert new rows into tables.
        UPDATE           tables, columns            Allows users to modify values in existing table rows.
        DELETE           tables                     Allows users to delete existing table rows.
        INDEX            tables                     Allows users to create and drop indexes on particular tables.
        ALTER            tables                     Allows users to alter the structure of existing tables
        CREATE           databases, tables          Allows users to create new databases or tables. 
        DROP             databases, tables          Allows users to drop (delete) databases or tables.

        GRANT 权限：以WITH GRANT OPTION选项给出。


管理员的权限

        Privilege                Description
        CREATE TEMPORARY TABLES  Allows an administrator to use the keyword TEMPORARY in a CREATE TABLE statement.
        FILE                     Allows data to be read into tables from files and vice versa.
        LOCK TABLES              Allows the explicit use of a LOCK TABLES statement.
        PROCESS                  Allows an administrator to view server processes belonging to all users.
        RELOAD                   A~s an adm to reload grant tables and flush privileges, hosts, logs, and tables.
        REPLICATION CLIENT       Allows use of SHOW STATUS on replication masters andslaves. 
        REPLICATION SLAVE        Allows replication slave servers to connect to the master server. 
        SHOW DATABASES           Allows a list of all databases to be seen with a SHOW DATABASES statement.
        SHUTDOWN                 Allows an administrator to shut down the MySQL server.
        SUPER                    Allows an administrator to kill threads belonging to any user.


特别的权限

        Privilege                Description
        ALL                      所有用户和管理员权限，可写成ALL PRIVILEGES
        USAGE                    不授予权限，创建用户并允许他登录，可以以后再授予更多权限


REVOKE 命令

        从一个用户收回权限
                REVOKE  privileges  [( columns )]
                ON  item
                FROM  user_name


        撤销所有权限，以及WITH GRANT OPTION子句所赋予的权限
                REVOKE ALL PRIVILEGES, GRANT
                FROM  user_name


查看数据库中具体某个用户的权限

show grants for 'user_name'@'%';
select * from mysql.user where user='user_name' \G;
