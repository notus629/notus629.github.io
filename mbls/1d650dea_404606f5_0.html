Difference between PEAR and Composer? [1]

    PEAR is used to install a PHP software package and Composer is used for the same thing.

    It might be a bit soon to regard PEAR as deprecated but I don't believe it will take long before it is. Composer is thriving and mature with a lot of packages and and active development. Therefore it might be better to use Composer as the standard for package management tool in PHP.



PEAR 缺点 [2]

    PEAR: PHP Extension and Application Repository

    依赖处理容易出问题

    配置非常复杂
 
    难用的命令行接口



Pyrus

    Pyrus PEAR2's installer

    示例：

        ## pear 安装 Mail_Mime 扩展
        $ pear install Mail_Mime

        ## Pyrus
        $ php pyrus.phar install pear/Mail_Mime


PECL [3]

    PECL is a repository for PHP Extensions, providing a directory of all known extensions and hosting facilities for downloading and development of PHP extensions.

    The packaging and distribution system used by PECL is shared with its sister, PEAR.



编译共享的 PECL 扩展 (shared PECL extensions) 的两种方式

    使用 pecl 命令

    使用 phpize
