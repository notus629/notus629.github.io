RewriteRule - Rewrite Conditions

    One or more RewriteCond directives can be used to restrict the types of requests that will be subject to the following RewriteRule.  


    The first argument is a variable describing a characteristic of the request,

    The second argument is a regular expression that must match the variable,

    A third optional argument is a list of flags that modify how the match is evaluated.




Examples

    ## send all requests from a particular IP range to a different server,
    RewriteCond "%{REMOTE_ADDR}" "^10\.2\."
    RewriteRule "(.*)" "http://intranet.example.com$1"

    ## When more than one RewriteCond, they must all match for the RewriteRule
    ## deny requests that contain the word "hack" in their query string, 
    ##   unless they also contain a cookie containing the word "go"
    RewriteCond "%{QUERY_STRING}" "hack"
    RewriteCond "%{HTTP_COOKIE}" "!go"
    RewriteRule "." "-" [F]


    ## direct the request to a different directory depending on the hostname used to access the site
    ## If the request was for http://example.com/foo/bar, then %1 would contain example.com and $1 would contain foo/bar.
    RewriteCond "%{HTTP_HOST}" "(.*)"
    RewriteRule "^/(.*)" "/sites/%1/$1"
