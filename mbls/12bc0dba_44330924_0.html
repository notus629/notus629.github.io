预备知识

    在使用 SSH 访问时，SSH 客户和 SSH 服务器各自都需要有一对密钥对（公钥）。

    客户访问服务器时，会提示服务器的 SSH 公钥证书不可信之类，就是指服务器的公钥。

    而客户要想不使用密码访问服务器的话，须先把自己的 SSH key 公钥放到服务器上，这就是 ssh key 的添加。



生成 SSH 密钥对及添加 SSH key 到 GitHub

    对于 msysgit, 有命令 ssh-keygen, 选项 --help 可以查看帮助


    1. 查看密钥

        ## 进入密钥存放目录，可以查看是否已经有密钥文件了
        $ cd ~/.ssh


    2. 生成密钥

        ## -t 表示加密类型，也可不选
        ## -C 表示注释
        ## 会在当前目录下生成密钥对
        $ ssh-keygen -t rsa -C "hehe_xiao@qq.com"    

        ## 或简化版，直接生成密钥对
        $ ssh-keygen


    3. 生成过程中会提示存放目录（默认当前），是否要密钥，就是 ssh key 的密钥，一般可不设置（更加方便）。最后得到两个文件：id_rsa, id_rsa.pub

        
    4. 上步生成的 id_rsa.pub 即为公钥，应放置到 GitHub 上去

        打开 id_rsa.pub 文件, 复制其内容到 GitHub 相应位置即可。
