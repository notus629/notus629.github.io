3.  php中定义常量时，const与define的区别？

A.  区别有如下几点：

        a. const 用于类成员变量的定义，一经定义，不可修改。define不可用于类成员变量的定义，可用于全局常量。

        b. const可在类中使用，define不能。

        c. const不能在条件语句中定义常量, define可以。

        d. const采用一个普通的常量名称，define可以采用表达式作为名称。

        e. const只能接受静态的标量，而define可以采用任何表达式。

        f. const定义的常量时大小写敏感的，而define可通过第三个参数（为true表示大小写不敏感）来指定大小写是否敏感。

                如：define('FOO', 'BAR', true);   //不敏感

A.  The CRLF \r\n, should be used according to the php documentation and RFC 2822.  

        Since \r\n is native to Windows platforms and \n to Unix, you can use the PHP_EOL constant on Windows, which is the appropriate new line character for the platform the script is currently running on.

9. PHP 中的相对路径的问题?                                                                    

A. 
    原理分析
        PHP 中的相对路径中的当前路径，通常是指执行脚本所在目录，而不是语句所在脚本的目录。

        比如:
            路径 project/images/a.gif, project/test1/test1.php, project/test.php
            test.php 中有语句 echo "<img src=\"../a.gif\" />";
            在 test.php 中 include 'test1.php'， 并执行 test.php;

        则会出现找不到 a.gif 的错误，因为当前路径为 test.php 所在的 project 目录，而非 test1.php 所在的 project/test1 目录

    带来问题：若同一个文件如 test1.php 在多个文件中 include, 并且执行，则相对路径无法统一。

    解决方案：

        php 现有的路径常量大多数都是与当前执行脚本相关，会变化，无法满足需求，__FILE__常量包含了语句所在的原始文件的全路径，但是操作系统的路径，不能用于 html 中的路径。

        因而，现有解决方案为：使用绝对路径，编写绝对路径函数来解决此问题，详见参考代码页。

    注意事项：好像 include 语句中的文件相对路径并不受相对路径影响（未证实）
