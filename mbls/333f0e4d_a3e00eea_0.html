推送至远程仓库

    即，将本地仓库的内容推送到 GitHub 上的远程仓库


    基本步骤

        1. 在 GitHub 上创建一个远程仓库，最好与本地仓库同名，如 testrepo

        2. 利用 git remote add, 将 GitHub 上的仓库设置为本地仓库的远程仓库

        3. 在本地利用 git push 将本地仓库推送到 GitHub 上的远程仓库



在 GitHub 上创建远程仓库

    创建时最好不要勾选，README, ignore, license 等，方便与本地仓库整合

        即使之前有内容，然后全部删除了也不行，因为仓库里有记录，必须要是一个全新的未 commit 过的空仓库

        否则本地仓库的内容 push 到 GitHub 时会冲突


    GitHub 上创建的仓库路径： git@github.com:用户名/仓库名.git



在本地将 GitHub 上的仓库设置为本地仓库的远程仓库

    ## origin 为该远程仓库在本地的标识符，习惯设置为 origin
    ## 创建好本地仓库的远程仓库后，就可以利用 git push 进行推送
    $ git remote add origin git@github.com:notus629/testrepo.git
