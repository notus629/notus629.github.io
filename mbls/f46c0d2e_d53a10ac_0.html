总结

    1. 用 word 或 pages 制作, 导出为 pdf（考虑不同平台的阅读体验）。另外 word 格式也可以。


    2. 推荐的命名方式，提供小许信息，如姓名、职位、文档说明

        举例：李小明_Android工程师_简历.pdf


    3. 标准的 A4 纸尺寸、适当的页面边距（更好的阅读体验或适当增加页面的可阅读内容）

 
    4. 控制页数为2~3页，不添加标题或者姓名作为标题

    
    5. 避免打印时会出现空白或只有半页的情况，不要制作简历封面


    6. 不使用照片或使用较正式的证件照，不使用招聘平台生成的简历（至少要修改）


    7. 不要给简历增加背景图（阅读效果变差），使用标准字体，绝不允许出现重复的内容。内容太多时，要增加页码（超过2页时加上）。


    8. 避免口语话，不要轻易使用精通二字，也不能都写成了解，注意用词。

   
    9. 不要写任何带有抱怨、负面的词语。尽量不要用“认为”、“觉得”、“应该”之类带有不确定语气的词语（使人感觉不自信）

   
    10. 常用名词拼写规范（让人觉得更严谨）,大小写要规范，如

        Java （非 java, JAVA等）

        其它举例：Linux, App, Runtime, HTTP, HTML, CSS, JavaScript, Git, Eclipse, MySQL, SQLite, Node.js, XMPP, JSON, XML, jQuery, Ruby, Python, Web, Docker, Mac OS X, Atom
    

    11. 不要随便使用或更改缩写，以官方名称为准，如 ThinkPHP 缩写为 TP

    
    12. 只使用一种日期格式，如 2017年2月3日（扩展一下，其它格式也要统一）


    13. 个人资料

        尽量不使用 QQ 邮箱（特指数字@qq.com形式的）。

        不要写家庭地址（毫无意义）

        不要写目前行业和目前岗位（若目前行业不是互联网、不是相应岗位会大大扣分）

        专业信息与教育经历的专业信息重复

        个人博客链接地址过长过于复杂（可以用个人域名做跳转）


    14. 工作经历

        倒序排列工作经历（最近的最上面，同理，其它内容也倒序排列）

        写上在职时间（长时间无业、跳动过于频繁需慎重处理)

        不要写薪资，也不要写为保密（太低太高都有不好）

        写明职位职责（用于HR判断你的经验和当前招聘岗位是否合适）

        不要将职位职责写成项目职责或技能列表

        职位职责过于笼统


    15. 项目经历

        不要只写项目名称和时间，不描述其它细节

        项目简介篇幅不要过长

        要写明自己在项目中的职责

        写明项目时间，不要写开发周期

        写明项目为团队开发或个人开发

        已上线的应用建议提供短链接

        写明项目职责和关键技术点

        
    16. 简历投递

        检查发件人的名称，避免出现对方显示的发件人是个非主流的名称

        标题和正文：标题不要为空，正文写些客套话，不建议直接把简历写到正文里

        加分项：使用Git, 使用Markdown, 参加或发起过开源项目（使用Github）,坚持写博客


    17. 其它.

        预先准备好演示作品
