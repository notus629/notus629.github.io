问题描述：chrome能访问127.0.0.1，但无法访问localhost



解决方案：(status:solved, Leo, 2016-11-4)

    This is an IPv6 problem. Google must have just updated Chrome.

    1. First of all ensure that your hosts file has the following line and that it is uncommented.
        hosts file located at : C:\Windows\System32\drivers\etc\ hosts

        ::1 localhost


    2. Next, open up your Apache config (httpd.conf) and add the following to the listen section:

        Listen [::1]:80


    3. Next, you need to edit you Directory statements in httpd.conf or your vhosts files. They probably look something like this.

        <Directory "C:\path">
            Options Indexes FollowSymLinks
            Order Deny,Allow
            Deny from all
            Allow from 127.0.0.1
            AllowOverride All
        </Directory>

        Add an extra line after the 'Allow from 127.0.0.1' so it looks like this

        <Directory "C:\path">
            Options Indexes FollowSymLinks
            Order Deny,Allow
            Deny from all
            Allow from 127.0.0.1
            Allow from ::1
            AllowOverride All
        </Directory>
