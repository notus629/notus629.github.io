原因

    PHP 出现错误，但是没有输出错误，因此显示为空白

    在 firefox 里头显示 http 500 服务器内部错误




打开错误提示

    php.ini

    display_errors = off 改为 on

    如下示例，会生成错误，否则一片空白，无输出，无错误
        <?php
            ini_set('display_errors', "On");
            error_reporting(E_ALL & E_NOTICE);
            echo "he";;lkdsjlafjdsalkfjdsal
        ?>


php.ini 位置

    在 ubuntu 16.04 LTS 中以 apt-get install php7.0 的形式安装

    /etc/php7.0/apache2/php.ini
    /etc/php7.0/cli/php.ini
    /etc/php7.0/pfm/php.ini
    /etc/php7.0/模块/模块名.ini

    应该修改 apache2下的php.ini，但实际没有效果。


    

解决方案

    注释掉declare(encoding='UTF-8');

    就不会再显示空白，可能是异常，但是又没显示错误。

    这个在php5.2里是没问题的，在php7.0里才出现了错误。



PHP document

    Note that in PHP 7 <?php declare(encoding='...'); ?> throws an E_WARNING if Zend Multibyte is turned off.
