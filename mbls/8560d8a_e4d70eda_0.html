memcached session 与传统 session 比较

    memcached 主要的目的是提速，因此它是一种无状态的数据。即数据不和用户绑定

    session 数据是和用户是绑定的，因此是一种有状态的数据



memcached 安全机制讨论

    memcached 本身没有用户名密码机制，可以直接访问


    Windows 下

        可以将 memcached 放在内网, 用防火墙限制 memcached 的11211端口被访问


    Linux 下

        也可以启用防火墙

        setup 配置防火墙

        用 iptables -a input -p tcp -s 127.0.0.1 -dport 11211 -j ACCEPT

        
适合放在 memcached 的数据

    变化频繁，具有不稳定性的数据，不需要实时入库（比如用户在线状态、在线人数..)

    门户网站的新闻等，觉得页面静态化仍不能满足要求，可以放入到 memcache 中。(配合 jquery 的 ajax 请求）



不适合放在 memcached 的数据

    重要的数据，特别巨大的数据（如电影、视频等）

    除适合放在 memcached 的数据外的其它数据
