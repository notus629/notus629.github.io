错误信息如下：

    [Err] 1055 - Expression #1 of ORDER BY clause is not in GROUP BY clause and contains nonaggregated column 'information_schema.PROFILING.SEQ' which is not functionally dependent on columns in GROUP BY clause; this is incompatible with sql_mode=only_full_group_by



解决方法：

    在 /etc/my.cnf 文件里加上如下：(windows 为 my.ini)

        sql-mode='STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION'

        默认是开启了 'ONLY_FULL_GROUP_BY'，去掉即可


    然后，重启Mysql服务即可。



注：

    实际尝试多次失败。

    1. hmailserver 依赖于 mysql ，导致用 wampserver 重启时，实际上没有重启成功。 

    2. 网上写的是 sql_mode, my.ini的注释里为 sql-mode，不知道是不是都可以，反正最后改成了: sql-mode.

    3. 版本为 mysql 5.7.14, 似乎是 5.7 之后，才默认为 ONLY_FLL_GROUP_BY 模式

    4. 虽然会报此错误，但实际语句中没有使用 ORDER BY，而且最后该执行的语句全部都得到了该要的结果。
