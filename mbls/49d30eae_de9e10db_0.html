git clone - 获取远程仓库




clone 项目到本地

    ## 会将项目名（GitHub 上的 Repository name）作为文件夹，里面包含 git 仓库(即 .git 隐藏文件夹）和项目文件
    ## 下载到 git bash 当前目录，提示输入 ssh key 的密码
    ## clone 后会默认处于 master 分支下，系统会自动将 origin 设为该远程仓库的标识符
    ## 只会获取 master 分支到本地
    $ git clone git@github.com:github_username/repository_name.git
    
    ## -a 参数，(all) 查看当前分支的相关信息，同时显示本地仓库和远程仓库的分支信息
    $ git branch -a



获取远程的特性（feature-N）分支

    ## 在本地创建并切换到 feature-N 分支，而该分支的来源为 origin/feature-N
    $ git checkout -b feature-N origin/feature-N



向远程分支推送本地 feature-N 的修改

    ## 本地对 feature-N 进行了修改，将此修改推送给远程的 feature-N 分支
    ## 因为前面在 git checkout -b ... 时已经设置好对应的上游分支，因而直接 git push 即可
    ## 本地当前分支为 feature-N
    $ git push
