存储引擎

    即表格类型，每个表可以使用不同的存储引擎
        TYPE= type            // 旧的声明方式， 已在新版本中弃用
        ENGINE = type         // 新的声明方式

    常见的表格类型
        MyISAM
            默认类型，即有索引的顺序访问方法(Indexed Sequential Access Method)
            可被压缩，支持全文搜索，不是事物安全的，不支持外键。
        InnoDB
            事务安全的，提供COMMIT和ROLLBACK功能，支持外键，比MyISAM慢



事务安全性的 4 个需求（ACID）

    原子性(Atomicity)
    一致性(Consistency)
    孤立性(Isolation)
    持续性(Durability)



自动提交（autocommit)

    默认 MySQL 是以自动提交模式运行，所执行的语句都将立即写入数据库（提交）中

    关闭自动提交：set autocommit=0;



通过 InnoDB 使用事务

    若自动提交被打开，开始事务
        set transaction;

    若自动提交被关闭，输入一个 SQL 语句，一个事务将自动启动。
        commit;        --使用完后提交
        rollback;      --回到之前的状态
