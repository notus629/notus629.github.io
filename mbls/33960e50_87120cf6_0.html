Wampserver 成功安装记录 (2016-12-2)

    The MSVC runtime libraries VC9, VC10, VC11 are required for Wampserver 2.4, 2.5 and 3.0, even if you use only Apache and PHP versions with VC11.

    Runtimes VC13, VC14 is required for PHP 7 and Apache 2.4.17 or more

    Make sure you are "up to date" in the redistributable packages VC9, VC10, VC11, VC13 and VC14

    安装路径中不要有空格，不允许安装在 fat 文件系统之上


    安装后解决的错误：

        失败问题：将 vc9, vc10, vc11, vc12, vc14 的 redistributable 包安装之后，“apache httpd的无法定位程序输入点...libpruti”



失败安装记录，2016-11-20

    环境：Wampserver3.0.6, windows7 32bit in VirtualBox 5.0.20

    错误：The program can't start because VCRUNTIME140.dll is missing from your computer.


    vcruntime140.dll

        This files most often belongs to product Microsoft® Visual Studio® 2015 Preview. and were most often developed by company Microsoft Corporation. This files most often have description Microsoft® C Runtime Library. This file is Dynamic-link Library. This library can be loaded and executed in any running process.


    错误解决方法

        VCRUNTIME140.dll 是使用了VS库的软件必须依赖的动态库，此版本包含在vs2015中，利用微软的 Visual C++ Redistributable for Visual Studio 2015 安装包 （vc_redist.x64.exe/vc_redist.x86.exe）可以安装相应的库。


    (未解决)安装vc_redist.x86.exe错误，（Microsoft Visual C++ 2015 Redistributable Setup Fails [0x80240017]）

        Method 1: perform a clean boot in Windows

        Method 2: Make sure that the date and time are correct on the computer.

        Method 3: Try to remove the temp file:
            a. Press Windows key + "R" to open RUN box
            b. Type %temp% and click on "OK".
            c. In temp folder location select all files and delete.

        Method 4: Try to reregister the Windows Installer service:
            a. Press Windows key+R.
            b. Type msiexec /unregister and press enter, it will not show anything.
            c. Again type in the Run box, msiexec /regserver and press enter, it will register but not show anything.
