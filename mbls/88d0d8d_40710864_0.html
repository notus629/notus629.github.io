.gitignore 配置

    一般来说每个 Git 项目中都需要一个 “.gitignore” 文件，这个文件的作用就是告诉 Git 哪些文件不需要添加到版本管理中。放在项目根目录下

    Win7 及之后，直接新建 .gitignore. 文件，windows 会自动忽略最后的. 或用命令行来新建, c:\>dir > .gitignore

    安装了 msysgit 后，直接 $touch .gitignore

    Git 对于 .ignore 配置文件是按行从上到下进行规则匹配的，意味着如果前面的规则匹配的范围更大，则后面的规则将不会生效；



配置基本语法

    以 '#' 开始的行，被视为注释    

    以斜杠 “/” 开头表示目录；

    以星号 “*” 通配多个字符；

    以问号 “?” 通配单个字符

    以方括号 “[]” 包含单个字符的匹配列表；

    以叹号 “!” 表示不忽略(跟踪)匹配到的文件或目录；



基本示例

    ## 忽略掉所有文件名是 foo.txt的文件
    foo.txt    


    # 忽略所有生成的 html文件
    *.html


    # 忽略所有.o 和 .a 文件
    *.[oa]
