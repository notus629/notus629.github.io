.gitignore 配置

    一般来说每个 Git 项目中都需要一个 “.gitignore” 文件，这个文件的作用就是告诉 Git 哪些文件不需要添加到版本管理中。放在项目根目录下

    Win7 及之后，直接新建 .gitignore. 文件，windows 会自动忽略最后的. 或用命令行来新建, c:\>dir > .gitignore

    安装了 msysgit 后，直接 $touch .gitignore

    Git 对于 .ignore 配置文件是按行从上到下进行规则匹配的，意味着如果前面的规则匹配的范围更大，则后面的规则将不会生效；



配置基本语法

    以 '#' 开始的行，被视为注释    

    以斜杠 “/” 开头(结尾？？)表示目录；

    以星号 “*” 通配多个字符；

    以问号 “?” 通配单个字符

    以方括号 “[]” 包含单个字符的匹配列表；

    以叹号 “!” 表示不忽略(跟踪)匹配到的文件或目录；



基本示例

    ## 忽略掉所有文件名是 foo.txt的文件
    foo.txt    


    # 忽略所有生成的 html文件
    *.html


    # 忽略所有.o 和 .a 文件
    *.[oa]


    # 此为注释 – 将被 Git 忽略
    *.a       # 忽略所有 .a 结尾的文件
    !lib.a    # 但 lib.a 除外
    /TODO     # 仅仅忽略项目根目录下的 TODO 文件，不包括 subdir/TODO
    build/    # 忽略 build/ 目录下的所有文件
    doc/*.txt # 会忽略 doc/notes.txt 但不包括 doc/server/arch.txt



Git 忽略 .gitignore 原因 - PHPStorm 中的 .idea 目录无法忽略


    网友建议 $ git rm -r -f --cached .idea， 加上一个 -f 就可以强制删除解决这个问题（待实践 [3], 2017-6-1，Notus)


    1. .gitignore只能忽略那些原来没有被track的文件，如果某些文件已经被纳入了版本管理中，则修改.gitignore是无效的

    2. git 的本地忽略设置必须保证 git 的远程仓库分支上没有这个要忽略的文件，如果远程分支上存在这个文件，本地在设置 ignore 这个文件，将会失败，无法 commit 忽略。[3]
    

    1. 的解决方案

        ## 先把本地缓存删除（改变成未track状态），然后再提交
        $ git rm -r --cached .
        $ git add .
        $ git commit -m 'update .gitignore'

    2. 的解决方法

        删掉 .deal 目录，提交，再 git rm , git add & commit

        再 git status 就正常了（不会提示 .idea 目录下的改动）
